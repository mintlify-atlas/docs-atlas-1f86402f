## What is SentimentAPI?

SentimentAPI is an AI-powered web application that lets your team analyze the sentiment of customer comments — one at a time or in bulk via CSV upload. Each comment is classified as **POSITIVO** (positive) or **NEGATIVO** (negative) with a probability score so you know how confident the model is.

The platform is designed for operations and customer-experience teams that need to act fast on large volumes of feedback without building their own ML pipeline.

## How it works

<Steps>
  <Step title="Log in to SentimentAPI">
    Authenticate with your email and password. A secure JWT session is created that expires after 2 hours.
  </Step>
  <Step title="Analyze a comment or upload a CSV">
    Use the **Analysis Panel** to enter a single comment, or go to **Bulk Analysis** to upload a CSV file with many comments.
  </Step>
  <Step title="Review results">
    Each comment is returned with a `prevision` field (`POSITIVO` or `NEGATIVO`) and a `probabilidad` score between 0 and 1.
  </Step>
  <Step title="Export your data">
    Download the results as a CSV file directly from the dashboard for offline use or further analysis.
  </Step>
</Steps>

## Sentiment result structure

Every analysis — whether single or bulk — returns results in the same format:

```json
[
  {
    "texto": "The service exceeded my expectations",
    "prevision": "POSITIVO",
    "probabilidad": 0.964
  },
  {
    "texto": "Delivery was very slow and packaging was damaged",
    "prevision": "NEGATIVO",
    "probabilidad": 0.921
  }
]
```

| Field | Type | Description |
|---|---|---|
| `texto` | `string` | The original comment text |
| `prevision` | `"POSITIVO"` \| `"NEGATIVO"` | Model classification |
| `probabilidad` | `number` | Confidence score between 0 and 1 |

<Note>
  All API endpoints require a valid JWT Bearer token in the `Authorization` header. Obtain your token by logging in via `POST /login`.
</Note>
